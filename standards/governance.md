# 基线治理

## 所有权

将基线引入团队仓库前，为 `.github/instructions/`、`.github/skills/`、`.github/agents/`、`.baseline/hooks/` 与 CI 工作流指定 Code Owner。

## 变更策略

- 所有生效基线变更都必须经过 PR、审查证据和相应 Code Owner 批准。
- 新 Guide 先作为实验能力，并至少在一个低风险服务中试点。
- 规则必须简洁、精确，并能关联到真实故障模式或合规要求。
- 重复、过期或已被机械化门禁覆盖的规则，应通过同一审查流程删除。

## 信号策略

信号只保存枚举原因码和聚合检查结果，不保存源码片段、提示词、用户消息、标识符、密钥、URL、工单或客户数据。
