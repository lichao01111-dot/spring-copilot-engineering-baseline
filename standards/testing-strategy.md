# 测试策略与真实性标准

## 测试层级

| 层级 | 验证对象 | 何时必须使用 | 不适合替代 |
|---|---|---|---|
| 单元测试 | 领域规则、状态机、纯转换、错误分支 | 任何业务规则 | Spring 配置、SQL、真实序列化 |
| Web 测试 | 请求映射、校验、错误响应、认证入口 | HTTP 契约变更 | 服务层授权、真实数据库语义 |
| 集成测试 | Spring 装配、事务、JPA、migration、消息 | DB/MQ/安全/配置变更 | 大规模性能证明 |
| 契约测试 | 对外 API/事件兼容性 | 消费者或公开契约变更 | 业务逻辑完整性 |
| 制品/冒烟测试 | 打包后启动、配置、探针 | 发布前 | 功能边界覆盖 |

## 真实性判据

- 缺陷测试必须在修复前可失败，或清楚说明为什么无法回放旧版本。
- 对于授权测试，至少包含同组织允许、跨组织拒绝、无身份拒绝和管理员例外（如有）。
- 对于并发/幂等测试，验证最终持久化状态和事件数量，不能只看 HTTP 200。
- 对于异常测试，验证稳定错误码与无敏感泄露，而不是只断言抛出了任意异常。
- 对于 Mock，优先模拟不可控边界；不要模拟本应由真实数据库/框架验证的行为。
